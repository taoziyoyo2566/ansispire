# EDA Implementation Plan: Lightweight Reaction Engine (2026-04-16)

## Background & Motivation
The Ansispire project needs an Event-Driven Automation (EDA) mechanism that triggers automated responses (e.g., self-healing, alerting) based on audit events, without violating our ~200MB RAM footprint.

## Proposed Solution (Lightweight Reaction Engine)
1. **Rule Definition**: Introduce `extensions/eda/rules.json`.
2. **Reaction Engine**: Create `controller/audit/reactor.py` to tail JSONL logs.
3. **Execution Actions**: Webhook (alerting) and Shell (healing).

## Phased Implementation Plan
1. **Phase 1**: Rule Engine Core (`reactor.py`).
2. **Phase 2**: Action Handlers (Webhook/Shell).
3. **Phase 3**: Sample Rulebook (`rules.json`).
4. **Phase 4**: Documentation Sync.
