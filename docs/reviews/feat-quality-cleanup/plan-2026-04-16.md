# Quality Cleanup Plan (2026-04-16)

## Objective
To bring the entire Ansispire repository (including external dependencies) up to `production` profile standards in `ansible-lint`. This resolves over 100 linting violations and enforces consistency.

## Key Files & Context
- `roles/common/`
- `roles/webserver/`
- `roles/geerlingguy.docker/` (external role chosen for local fixing)
- `examples/advanced_patterns.yml`
- `controller/semaphore/bootstrap.yml`
- `.ansible-lint`

## Implementation Steps
1. **FQCN Adoption**: Replace all short module names (e.g., `apt`, `service`, `file`) with Fully Qualified Collection Names (e.g., `ansible.builtin.apt`) across all tasks.
2. **Variable Prefixing**: Prepend `common_` to variables registered inside `roles/common/`, `webserver_` for `roles/webserver/`, etc.
3. **YAML Formatting**: Fix `---` document starts, comment indentations, line lengths, and commas formatting.
4. **Jinja Spacing**: Correct all spacing issues in Jinja templates (e.g., `{{var}}` to `{{ var }}`).
5. **Task Naming & Keys**: Name all unnamed tasks, fix key ordering (name, tags, block, rescue, always), and ensure capitalization matches standards.
6. **External Role Refactoring**: Manually fix all lint errors inside `roles/geerlingguy.docker/`.

## Verification & Testing
- Run `make lint` and ensure 0 violations.
- Run `make syntax` to ensure playbooks are syntactically valid.
