# Plan — Fix Deps & CI Hygiene (2026-05-13)

> **Retroactive plan note**: This work emerged organically from a user
> question about CI log warnings; there was no pre-written plan. This doc is
> a retroactive plan to satisfy workspace W-R "plan + changelog" landing
> criteria. The round changelog (`round1-2026-05-13.changelog.md`) is the
> authoritative record of what shipped.

## Purpose

Eliminate misleading-but-non-fatal noise in CI logs, align the Python
dependency manifests (`requirements.txt` ↔ `pyproject.toml`) with the
project's stated SSOT and "minimum-not-pin" policy, restore `pip install -e .`
as a working install path, and update Dependabot to target the
development-phase trunk (`dev`).

## Level Declaration

**[L1] Engineering** — single-component refactor surface (CI config + dep
manifests + one feature-map doc). No architectural shift; no NFR change; no
new public interface.

Review dimensions per workspace §1-B: correctness (yes — fixes broken pip
install), maintainability (yes — SSOT alignment reduces drift), community
alignment (yes — matches dependabot + hatchling conventions), safe defaults
(yes — `target-branch: dev` matches the actual workflow).

## Scope

In scope:

1. CI log noise — Molecule 26+ shared-state probe ("CRITICAL ... glob failed")
2. Dependabot configuration — `target-branch: dev` so PRs land where the
   actual trunk lives
3. Python dep manifest alignment — `requirements.txt` is SSOT, pyproject
   floors mirror it for overlapping deps
4. `pyproject.toml` installability — `hatchling.backends` typo + empty-wheel
   `bypass-selection`
5. Doc drift in `docs/reference/feature-map/` — CI section listed 3 jobs;
   actual is 6 (DAG with two stages + one independent job)
6. Memory artifacts — branching-model clarification (`dev` as
   development-phase trunk) and new version-policy memory

Out of scope (deferred):

- pyproject-only deps without a SSOT counterpart (yamllint, docker, pytest,
  ansible-navigator floors are unchanged)
- Adding requirements.txt-only deps (cryptography, requests, PyYAML) into
  pyproject — these are documented as transitives of ansible-core in
  requirements.txt; doubling them into pyproject is a separate decision
- Existing 6 OPEN Dependabot PRs against `master` (user-owned; recommendation
  is "close and let Dependabot regenerate against `dev` per the new config")
- Docker driver schema warning ("Driver docker does not provide a schema") —
  upstream limitation, not fixable on our side

## Judgment Criteria

- Each change passes the W-R15 three-gate verification loop (direction →
  syntax → functional test) in a single iteration.
- `pip install -r requirements.txt` and `pip install -e ".[test]"` both
  resolve cleanly against the updated manifests.
- All 4 real Molecule scenarios still pass `molecule syntax -s <X>` with the
  new `default` stub present.
- `yamllint .` against tracked files passes (CI parity).
- `ansible-playbook --syntax-check` passes against `stag` + `prod` inventories.
- Vault crypto path round-trips under cryptography 48.0.0 (smoke test for
  Dependabot PR #11's eventual regeneration).

## Decision Rationale (Per Change)

| Change | Rationale |
|---|---|
| Add stub `molecule/default/molecule.yml` | Molecule 26 probes the file at startup to set up shared-state; missing file → misleading `CRITICAL ... Exiting.` log. Source-read confirms shared-state is opt-in per scenario, so the stub does NOT enable it for the four real scenarios. |
| Sync `docs/reference/feature-map/` CI section to actual ci.yml | Doc said 3 jobs; reality is 6 with a two-stage DAG. Future agents would re-derive from ci.yml each time; cheaper to fix the doc. |
| `dependabot.yml: target-branch: dev` (both ecosystems) | Project is in development phase; `dev` is the trunk; `master` is reserved for post-launch. Default target (`master`) generates stale-base PRs (every existing OPEN PR currently shows YAML Lint FAILURE for this reason). |
| `requirements.txt: ansible-core==2.20.5 → >=2.20.5` | "Requirements declares minimums, not pins" (user-stated 2026-05-13). The `==` was the only `>=`-outlier in the file. |
| `pyproject.toml: ansible-core>=2.16,<2.18 → >=2.20.5` | Old cap excluded the pinned 2.20.5 (PEP 440 `<2.18`). New floor matches requirements.txt; cap dropped per "default latest stable" policy. |
| `pyproject.toml: build-backend = "hatchling.build"` + `[tool.hatch.build.targets.wheel] bypass-selection = true` | Typo `hatchling.backends` made `pip install -e .` fail; bypass-selection acknowledges this project ships no Python package source (deps-only manifest). Restores documented install path. |
| `pyproject.toml: align overlapping floors to requirements.txt` | SSOT discipline; pyproject was permissive on some (jmespath, ansible-lint), restrictive on others (molecule) — drift symptom. |
