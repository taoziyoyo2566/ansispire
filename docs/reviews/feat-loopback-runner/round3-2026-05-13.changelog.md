## 2. Review findings fixed

| Finding | Resolution |
|---|---|
| `quick` / `standard` required Docker despite being Docker-free gates | Docker preflight is now mode-scoped to `full` / `ci-equiv`. |
| `coverage --fail-under` was inflated by test files under `controller/audit/test_*.py` | Coverage report and threshold now use `--omit=controller/audit/test_*.py`; current target-code total is 73%. |
| `make verify` and runner standard were not equivalent | `verify` now includes lint, leak-scan, Python tests, and `dry-run`; runner calls the same named static gates. |
| CI secret detection used `detect-secrets scan --baseline`, which does not fail on new findings | CI now runs `python scripts/detect_secrets_gate.py`. |
| CI did not run the newly registered relay/sink/filter tests | CI now has a `python-tests` job running `make test-eda` and `make test-filters`. |
| Ansible used `~/.ansible/tmp`, which can fail in sandboxed / read-only-home environments | Make and runner now set writable Ansible temp dirs. |
