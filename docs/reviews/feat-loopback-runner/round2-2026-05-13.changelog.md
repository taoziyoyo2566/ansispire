## 2. Intent per change-group

| Group | Intent |
|---|---|
| Make targets + composite (`test-eda` chain) | Stop the "orphan test" anti-pattern: every test file now reachable from a named gate, so CI / runner / contributors all converge on the same SSOT. |
| `requirements.txt` floors for detect-secrets + coverage | Previously the runner did `pip install coverage` mid-flight, which fails in offline / locked-down environments. Now declared in the SSOT (testing-governance §9 fresh-clone parity). |
| `.secrets.baseline` + gate script | Promote the "no new secret leaks" check from "CI-only" to a local push gate (testing-governance §4.2 parity). Gate is intentionally a regen-and-diff rather than mutating-in-place so baseline edits stay reviewer-visible. |
| 3 new TSVS docs + INDEX update | testing-governance §7 mandates TSVS for any new functional test. The three new L1 tests had been running silently as "extras" — now they're first-class. |
| `verify` includes `detect-secrets` | Local push-gate ↔ CI parity; surfaces leaks before a stale baseline reaches the remote. |
| `.gitignore` additions | Prevent committing the runner's per-run dirs (≈100 MB after a few runs) and coverage artifacts. |
| README "Running tests" section | Contributors land on `make verify*` first, runner second — both routes from a single page. |
| testing-governance §3 + §4 sync | W-R12 sync-update: rules referenced from this file must remain accurate after Makefile changes. |
