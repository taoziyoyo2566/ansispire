## 1. Purpose

Replace Gemini-authored `scripts/loopback_test_runner.sh` (v1) with a hardened v2 that:

1. Stops silently overwriting working-tree credentials (`.vault_pass`, `inventory/local/vault.yml`).
2. Stops the "EXCELLENT: 100% coverage" false-positive (v1 used `grep "100%"` which matched any module hitting 100%; true total was 86%).
3. Aligns the runner with the project's existing testing governance (testing-governance.md §3-§5) instead of reimplementing the gate logic alongside it.
4. Becomes reusable across sessions / branches as a structured, history-retaining pre-merge gate — not a one-shot script.

The v2 runner is "done" when:

- [x] `bash -n` clean.
- [x] `quick` mode passes end-to-end (~10s).
- [x] `standard` mode passes end-to-end and runs real `coverage --fail-under` check.
- [x] No file under the project tree is written/overwritten outside `test_results/`.
- [x] On failure, `SUMMARY.md` includes tails of every failed log.

## 4. Design decisions (rationale)

| Decision | Rationale |
|---|---|
| **Thin orchestrator over `make`** | SSOT stays in Makefile + testing-governance.md. Runner adds isolation/logging/aggregation, not gate redefinition. |
| **fail-collect not fail-fast** | A single run should surface the entire problem set, not require N round-trips. `make verify` remains fail-fast for hot-development use; runner serves the pre-merge "audit" role. |
| **Modes mapped to §5 gates** | `quick`/`standard`/`full`/`ci-equiv` = save-point/push/release/CI-mirror. One-line mental model for contributors. |
| **`MOLECULE_PARALLEL=0` default** | Matches testing-governance §4.2 "local serial / CI matrix parallel". Parallel is opt-in via env. |
| **Isolation dir via `mktemp`** | Eliminates the W-R13 (credentials-never-tracked) class of bug. Dummy vault, if requested, lives outside the tree. |
| **`coverage --fail-under`** | Replaces `grep "100%"` false-positive. Default threshold 70 = below current 87%, leaves headroom; configurable. |
| **Orphan tests run as "extras"** | Tests not in `make test-eda-*` (e.g., `test_filters.py`) are still picked up for coverage, but logged separately so the SSOT-drift remains visible — forcing future registration. |
| **`trap on_exit EXIT INT TERM`** | Guarantees teardown of control-plane + audit stacks on every exit path. Idempotent. |
| **History retention** | `test_results/run-<ts>/` + symlink + auto-prune to N=10. Lets contributors compare runs across branches without manual cleanup. |
