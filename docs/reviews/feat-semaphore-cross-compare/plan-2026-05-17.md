### D. Framework Best-Practice Pre-Check (W-R18)

| 方向 | 检查源 | 结论 |
|---|---|---|
| `SEMAPHORE_DB_PATH` 语义 | 上游 `deployment/docker/server/server-wrapper` 真实源码 | ✅ 目录语义，wrapper 自动追加 `database.sqlite` |
| Path A token 链路 | Ansible 模块文档 `ansible.builtin.stat` / `set_fact` 语义 | ✅ stat 注册后变量不变；`set_fact` 是规范做法 |
| Reactor 鲁棒性 | Python stdlib `urllib` retry pattern / threading docs | ✅ while-True + 显式 backoff 是 stdlib 范式 |
| Audit container baked-in tool | Docker best practice — base image vs runtime install | ✅ Dockerfile 烘焙工具是公认范式 |
| Healthcheck for relay/reactor | Docker compose `healthcheck` spec | ✅ 即使 stdin/stdout-only 服务也可用 `pgrep` |
| ACCESS_KEY_ENCRYPTION 持久化 | 上游 `https://semaphoreui.com/docs/.../security` | ✅ 推荐用 `head -c32 /dev/urandom \| base64`；持久化到 state dir |
| rules.json schema 校验 | extensions/eda/events.schema.json 既有模式 | ✅ 同类型 schema-as-code 范式 |
| API contract preflight | OpenAPI / Swagger 验证范式 | ✅ 与上游 `/swagger` 端点对接 |
| Reactor vs ansible-rulebook | Ansible 官方 EDA 路线 + 上游 `ansible.eda` collection | ⚠ 待 WU-5 IVG 给最终结论 |
| OSS Runner 抽象 | 上游 `/runners` 官方文档 + `semaphore runner register` CLI | ✅ 基础 Runner 是 OSS；Tag-routing 是 Pro |

**结论**：所有 WU-1 至 WU-4 的方向均已与上游或独立证据对齐；WU-5 本身就是为 WU-6/WU-7 答方向题。**Plan body 可以继续。**
