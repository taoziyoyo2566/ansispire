## Intent per change

- **Reactor truncation seek(0) not seek(EOF)**: This was the most serious bug — a data-loss bug in code that called itself "cursor persistence for zero event loss". The fix is the literal opposite of my WU-2 implementation. Lesson: when copy-truncate happens, the file is now smaller than the cursor — that's a SIGNAL that NEW content is coming from offset 0, not a signal to skip to whatever offset 0+new-file-size resolves to. I had the right detection logic but the wrong response.
- **Per-tick truncation detection**: My WU-2 only checked at startup. Logrotate's `copytruncate` mode runs WHILE the reactor is running — fd stays open, offset stays past the new size, `f.readline()` returns "" forever. Codex caught this; my fix matches his structurally.
- **Healthcheck `/proc/1/cmdline`**: `pgrep -f reactor.py` matches the grep process itself in some pgrep versions because `-f` matches the full argv (which contains "reactor.py" as the search term). PID 1 cmdline read is unambiguous.
- **Admin password API probe**: Real-world idempotency. Ansible's `changed` field is a signal to operators and downstream `notify:` handlers — reporting changed=1 every deploy when nothing actually changed pollutes that signal. Codex's gate accepts 400/401/403 as "needs change"; I matched that since both 401 (wrong password) and 400 (validation rejected the password against complexity rules) suggest the in-DB hash doesn't match what we want.
- **Audit role `build: always`**: Module's `build: policy` default = build-if-image-missing. After first deploy, subsequent deploys keep using the stale image even if `reactor.py` was edited. `build: always` forces rebuild every run; the Dockerfile is minimal so the cost is just a layer cache check.
- **Drop SEMAPHORE_ADMIN_PASSWORD**: Audit relay/sink/reactor authenticate to Semaphore via Bearer token. The admin password was passed via env into every audit-* container and never consumed — pure security smell removed.
- **Schema `_contains` string-only**: Substring match (`if str(value) not in actual_val`) is the reactor's behavior. For non-string values that becomes `"123" not in actual_val_str` — works, but semantically nonsense and likely a typo. Schema should reject.
- **Schema template identifier required**: reactor's `trigger_semaphore_task` POST payload contains `{"template_id": <int>}` — missing template means the API returns 400 at runtime. Schema must enforce at validation time.

## Retraction (Round 8, 2026-05-19)

Codex's second-pass review against `5c8df25` surfaced **four new defects**, two of which contradict claims in this Round 6 document:

1. **Line 13 retracted** — "Specifically refused: Codex's `ARG AUDIT_PYTHON_IMAGE` + `python:${AUDIT_IMAGE_TAG:-3.12-alpine}` pattern … Not adopted" is misleading. While the `ARG` form was indeed not adopted, the conflated `python:${AUDIT_IMAGE_TAG:-...}` pattern was *kept* at `controller/audit/docker-compose.yml:56` (relay) and at all three e2e python services. Setting `AUDIT_IMAGE_TAG=v0.1.0` on the cross-compare branch resolves the relay image to `python:v0.1.0` (nonexistent on Docker Hub). Codex's direction — disentangle the baked-image tag from the python base tag — was correct; the Round 6 refusal was a misjudgment. Fixed in Round 8 via a new `AUDIT_PYTHON_BASE_TAG` variable (see `round8-2026-05-19.changelog.md`).

2. **"No regressions" verdict at the bottom of the Re-verification section is over-broad.** The re-verification covered the 6 fixes themselves but did not exercise:
   - cross-restart cursor-zero ambiguity (`load_cursor()` returns 0 for both "missing" and "present == 0" cases — a reactor crash immediately after `save_cursor(0)` would skip the post-rotate file on the next boot)
   - per-rule exception isolation against non-dict rule entries (the v2.5 `except` block called `rule.get(...)` on whatever object `match_rule` rejected, re-raising AttributeError on non-dict rules)
   - `AUDIT_IMAGE_TAG` semantic split

   These three defects existed in `5c8df25` and were missed by the Round 6 re-verification gate plan. Round 8 lands the fixes and adds direct regression coverage. The Round 6 fixes themselves remain clean; the "No regressions" statement should be read as scoped to the 6 fixes, not as a global green-bar claim.

See `plan-round8-2026-05-19.md` + `round8-2026-05-19.changelog.md` for the corrective work.
