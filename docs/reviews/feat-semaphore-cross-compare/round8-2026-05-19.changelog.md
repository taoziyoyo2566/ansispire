## Intent per change

- **F1 (Optional[int] cursor)**: The data-loss bug is subtle but real — `save_cursor(0)` is the *correct* post-truncate marker, and a crash between writing the marker and reading the first post-rotate line would otherwise re-seek to EOF on next boot. The fix preserves the on-disk format (decimal integer text) and only changes the in-process interpretation: `None` means "file absent" (fresh boot → EOF), `int` means "honor this offset exactly" (0 → seek(0), >0 → seek to offset, > size → truncation detected). The existing `test_truncation_resets_cursor_to_start` test plants a non-zero cursor; the new `test_cursor_zero_means_seek_to_start_not_eof` test plants `"0"` to cover the exact failure mode Codex described.
- **F2 (AUDIT_PYTHON_BASE_TAG split)**: Two semantic dimensions deserve two variables. `AUDIT_IMAGE_TAG` keeps backward compatibility (operators who exported it stay green; manifest_sync remaps it to `audit_baked`). `AUDIT_PYTHON_BASE_TAG` is the new dedicated knob for the python base; defaults match (`3.12-alpine` everywhere) so there is no behavior change for anyone who does not opt in to a tagged release. Manifest-sync is the SSOT pipeline — extending its block (with idempotency under repeat runs) means a single `make manifest-sync` puts the host into the disentangled state.
- **F3 (isinstance guard)**: One bad rule must not kill the tail loop. The existing try/except catches *evaluation* failures (match_rule throwing on a bad condition); the new isinstance check at loop head catches *structural* failures (non-dict entry in rules array). Each block now does one job. Tests cover both paths.
- **F4 (Round 6 retraction)**: Per W-R13 pre-accept verification, and per the project value that incorrect statements on permanent record get retracted (not silently fixed), the Round 6 changelog gets an in-place retraction section with a forward-link to this Round 8 file. Future readers of Round 6 see the correction without having to cross-reference Round 8 separately.
