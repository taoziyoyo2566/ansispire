## 3. Intent per Change

### 3.1 IVG-EDA-RULEBOOK-MIGRATION
**问题**：自研 reactor（235 行 Python）vs upstream `ansible-rulebook` 的取舍
**方法**：核读上游官方文档 (sources / conditions / installation) + GitHub 仓库元数据；建立等价 YAML 翻译；量化运行时差异（JDK + jpy = ×5-8 内存、×4-5 镜像）
**结论**：**暂不迁移**。当前规模（2 条规则、单 hub、~1 job/min）下迁移成本远超收益。保留 4 项触发条件用于未来 re-evaluate。

### 3.2 IVG-EXECUTION-PLANE-RUNNER
**问题**：是否把 controller/executor 拆开（引入 Semaphore OSS Runner）
**方法**：核读上游 [Runners 文档](https://semaphoreui.com/docs/administration-guide/runners/) + [Configuration 文档](https://semaphoreui.com/docs/administration-guide/configuration/) + 上游 `services/runners/types.go` 源码（develop 分支）
**结论**：**暂不引入**。OSS 完全支持基础 Runner（tag-routing 是 Pro），但当前 fleet 规模（0 真实节点 + 1 job/min）下拆分无收益。强耦合 WU-3 (ACCESS_KEY_ENCRYPTION) 作为前置；保留 5 项触发条件 + 6-Gate 落地路径草案。

### 3.3 INDEX.md
追加 2 条登记行；位置在 IVG-SEMAPHORE-CROSS-COMPARE (2026-05-17) 之后，保持时间序列单调。
