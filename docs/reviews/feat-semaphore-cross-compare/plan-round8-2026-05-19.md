## §2. Decision Rationale

### F1 — Why Optional[int], not a separate flag
Adding a sentinel string or boolean ("HAS_CURSOR") to the cursor file format is a wire-compatibility break. `Optional[int]` is purely an in-process type change; the on-disk format (decimal integer text) is unchanged. The interpretation of "file present, contents == 0" becomes meaningful (= "we just learned about a truncation, start from offset 0"), while "file absent" maps to `None` (= "fresh boot, don't dump backlog").

### F2 — Why a new variable, not rename
- Renaming `AUDIT_IMAGE_TAG` would break any operator who set the env already; the cost of two parallel variables is low.
- Two variables also matches semantic reality: one tracks the Python base (Docker Hub `python:VERSION`), one tracks our published image tag (`ansispire/audit-*:VERSION`). They will diverge as soon as Ansispire ships its first tagged release.

### F3 — Why isinstance check at top of loop, not inside except
Putting `isinstance` at the loop head means `match_rule(payload, rule)` never gets called with a non-dict, so the `except` block does not need to handle the corrupt-rule-name case. Cleaner: each block does one job. The `except` continues to catch dict-rules whose internal evaluation throws (the original intent).

### F4 — Why retract Round 6 line 13 rather than silently fix
Two reasons: (a) Round 6 changelog claims "Not adopted" — that's a false statement of fact on permanent record; a future reader auditing the history needs to see the retraction. (b) W-R13 (pre-accept verification) is a project value; retracting our own misjudgment when re-verification catches it reinforces the value rather than papering over.
