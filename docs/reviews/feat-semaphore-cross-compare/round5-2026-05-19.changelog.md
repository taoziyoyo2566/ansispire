# Round 5 — WU-4 API Contract Preflight (changelog)

Reference: [`plan-2026-05-17.md`](./plan-2026-05-17.md) §4 WU-4.

Branch: `fix/codex-cross-compare-hygiene` (HEAD before this round: `1771d01` — WU-2 robustness).

## Intent

Land the cross-compare WU-4 work unit: a read-only API contract preflight
that surfaces upstream Semaphore schema drift BEFORE bootstrap.yml mutates
state, plus a CI gate against both the pinned and `latest` image tags.

Before this round, an upstream rename or status-code change would surface
mid-way through `bootstrap.yml` — leaving a partially-bootstrapped project
that the next run could not idempotently recover. After this round, the
preflight fails fast with a per-endpoint diagnostic, and CI catches the
drift at PR time (latest-tag matrix is `continue-on-error` so the project
gets advance notice without breaking unrelated PRs).

## Intent per change

- **bootstrap_preflight.yml**: separation of concerns — `bootstrap.yml`
  expects an API shape, `bootstrap_preflight.yml` codifies and verifies
  that contract. Two modes because the schema-mode probe is cheap enough
  to run before every bootstrap, but the full walk needs a throwaway
  project (mutates state) so it goes behind an explicit `-e` flag.
- **`import_playbook` at top of bootstrap.yml**: ensures the preflight
  runs as the first play. Variable scope works because `import_playbook`
  passes `-e` extras to both plays.
- **`preflight/` directory**: keeps the harness adjacent to its consumer
  (`bootstrap.yml`) instead of mixing with `audit/e2e/`, which is a
  different surface (reactor pipeline, not API contract). Two surfaces,
  two harnesses.
- **`compose.yml` minimal**: no audit stack, no shared `controller-net`.
  This is a pure API contract test, isolated by the dedicated `-p
  ansispire-apicontract` project label.
- **CI matrix [pinned, latest]**: the pinned tag is gating (must pass);
  `latest` is `continue-on-error` so upstream releasing a breaking change
  in `latest` doesn't block unrelated PRs but does surface as a warning
  in every PR run.
- **Image tag resolution priority in run.sh**: explicit env > manifest
  `semaphore_pinned` > `latest`. CI sets `SEMAPHORE_IMAGE_TAG=latest`
  only for the `latest` matrix entry; the `pinned` entry leaves it
  unset and lets run.sh read `config/manifest.yml`.

## NOT done in this round

- **Real `make test-api-contract` execution against `latest`** — needs
  Docker daemon access; user has paused destructive/host-impact operations.
  Local CI dry-run via `bash -n` + `yamllint -s` clean; full execution
  validated via `make verify` (lint/syntax) only.
- **DELETE /api/project/{id}** verification — best-effort cleanup
  (`failed_when: false`); if upstream returns 405 the throwaway project
  stays on the test container but the container is torn down anyway.
  Would only matter for long-lived test infra.
- **Negative tests** — preflight currently only validates the happy path
  (auth + shape). Future enhancement: rejected auth must return 401, not
  200 with empty body; missing fields in PATCH must return 400. Out of
  scope for WU-4 per plan §4.
- **Endpoint coverage gap** — `POST /api/users` (idempotent user create)
  and `POST /api/project/{id}/users` (RBAC bind) are NOT in the full-mode
  walk. They only matter for the Round 8 RBAC demo flow which has its
  own smoke test (`controller-rbac-smoke`). Adding them would double
  preflight runtime without commensurate coverage gain.
