## Intent

`ansible.posix.synchronize`'s failure surface when rsync is absent is opaque on both sides — remote returns the cryptic `rsync error: error in rsync protocol data stream (code 12)`, controller returns Ansible's generic `Failed to find required executable "rsync"`. Round 7 inserts a 4-task preflight (probe + assert × 2) that fails fast with an `apt`/`brew`/`apk` / `make hub-baseline` hint *before* any mutating step.

This is a UX improvement, not a correctness fix. The underlying `synchronize` task already fails when rsync is missing — it just fails badly. The preflight makes the failure self-diagnosing.

## Intent per change

- **`command: rsync --version` (not Codex's `stat /usr/bin/rsync`)**: probing the actual binary invocation handles macOS Homebrew (`/usr/local/bin/rsync` or `/opt/homebrew/bin/rsync`), Alpine, Debian, and any other layout without hard-coding paths. Also confirms the binary is *runnable*, not just present.
- **Controller-side probe with `delegate_to: localhost` + `become: false`**: rsync on the workstation is typically owned by the user, not root. The remote-side probe runs with the play's `become: true` automatically.
- **`failed_when: false` on probe + separate `assert`**: keeps the probe task silent in PLAY RECAP regardless of rc; the `assert` task is the place to put the actionable `fail_msg`. Cleaner than combining via `failed_when: rc != 0` because the failure message can be tailored to operator workflow (`make hub-baseline HUB_NODE=...` on remote vs `apt install rsync` on controller).
- **No Codex sudo-preflight (task 5)**: `deploy_hub.yml` sets `become: true` at the play level; the remote rsync probe runs *as the become user*, so if sudo is broken the probe fails with sudo's own (already clear) error. Adding a separate sudo-preflight is redundant.

```
=== POSITIVE CASE (normal PATH, rsync present at /usr/bin/rsync) ===
PLAY [localhost]
TASK [probe rsync]              ok
TASK [assert rsync present]     ok      (rc == 0 passes)
TASK [debug]                    ok      msg="OK rc=0 version=rsync version 3.2.7 protocol version 31"
PLAY RECAP   localhost: ok=3 changed=0 failed=0

=== NEGATIVE CASE (cmd: /nonexistent/rsync — simulates rsync absent) ===
PLAY [localhost]
TASK [probe rsync (simulated missing)]   ok      (failed_when: false absorbs rc=2)
TASK [assert rsync present]              FAILED  (assertion: "probe.rc == 0", evaluated_to=false)
                                                 msg="rsync missing on controller — install via apt/brew/apk"
PLAY RECAP   localhost: ok=1 failed=1
```

Negative case confirms the assert task fires with the actionable `fail_msg` exactly as designed. Positive case confirms the preflight is a no-op on a healthy controller.

All 3 gates clean in iteration 1 of 1. W-R17 cap not exhausted.
