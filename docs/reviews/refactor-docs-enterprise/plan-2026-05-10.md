## 1. Level Declaration

**[L2] Architecture** — touches every documentation surface, restructures `docs/` tree, renames foundational files (`SUMMARY.md`→`ARCHITECTURE.md`), and shifts contracts in `CLAUDE.md` §2 (lazy-loading order). Affects every future contributor and AI session.

Per project `CLAUDE.md` §1 + workspace `CLAUDE.md` §1-B / §2, this requires a written plan with user approval before implementation. This file is that plan.

## 2. Purpose

Bring documentation up to enterprise project standard. Concretely:

- **One entry, no ambiguity**: a newcomer should know in 10 seconds where to start and a specific subsequent path to take.
- **Audience-aligned organization**: documents grouped by the role of the reader (user / operator / developer / governance), not by the type of artefact.
- **No stale or load-bearing fictions**: kill `GETTING_STARTED.md` references to deleted paths; remove hard-coded versions in narrative prose; converge entry-point claims (currently 3 files all claim "authoritative entry").
- **Enterprise scaffolding**: `CHANGELOG`, `SECURITY`, top-level cleanliness.

## 3. Scope

### In scope

- README.md, SUMMARY.md, GETTING_STARTED.md
- All files under `docs/` (relocate, rewrite, or archive)
- `CLAUDE.md` §2 (lazy-loading order — the entry pointer changes)
- Top-level cleanliness (`loop_status.tmp`, `molecule_results.log`, `ANSISPIRE_*_REPORT.md` placement)
- New files: `CHANGELOG.md`, `SECURITY.md`

### Out of scope

- `controller/`, `roles/`, `playbooks/`, `extensions/` source code (only their per-directory `README.md` may be touched, and only to update doc cross-links)
- Inventory layout (already settled in commit `1761c9a`)
- Test infrastructure (`molecule/`, `tox.ini`, etc.)
- AI governance content (`CLAUDE.md` §0/§1/§3/§4 stays; `GEMINI.md` stays)
- `docs/reference-cn/snapshot-2026-04-14/` — translation snapshot, refresh deferred
- Investigations content (only `INDEX.md` routing column changes)

### Judgment criteria (acceptance)

A phase is **done** when:

1. The new structure / file is in place and renders correctly (no broken links via `find ... -name '*.md' | xargs -I{} grep -lE '\]\([^)]+\)'` cross-check).
2. CLAUDE.md / SUMMARY.md / README.md cross-references are all consistent (no orphaned mentions).
3. The "newcomer test" passes for each phase: a fresh reader can find the answer to a phase-specific question in ≤2 clicks from README.md.

The whole refactor is **landed** when:

- README.md ≤ 120 lines, contains exactly: positioning / capabilities / prereq / quickstart / docmap / governance.
- `docs/` has 4 audience-aligned subdirs each with an `INDEX.md` (or section in `docs/README.md`).
- A `CHANGELOG.md` exists with at least the recent 4 rounds extracted.
- No file under tracked git references a deleted path.
