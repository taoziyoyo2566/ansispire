## 1. Rubric recap

A doc passes a row if, for every scenario it claims to cover, a reasonably-skilled operator (Ansible-fluent, has read the README) can execute it without:

- (a) asking an AI for clarification
- (b) reading source code to figure out a variable / path
- (c) inferring a command that the doc references but does not spell out
- (d) cross-referencing 3+ other docs to assemble a procedure
- (e) following a step whose failure mode is unstated

Pass = ✅, partial = ⚠️, fail = ❌, N/A = —.

## 4. Backup / restore gap

**Confirmed absent.** Searched all 5 user-guide files: no section titled backup, restore, dump, snapshot, recovery. Closest related fragments:

- `03 §3 EDA token rotation` — delete + re-mint (this is "rotation", not "restore from backup")
- `02 §11.3 Vault 密码丢了` — recovery for lost vault password (not data backup)
- `02 §8.7 完全推倒重来` — destructive cleanup, opposite of restore

**What an unassisted operator currently cannot do**:

1. Back up the audit DB (`/var/log/semaphore/events.jsonl` + Semaphore postgres volume) before an upgrade or migration.
2. Back up `/var/lib/ansispire/state/` (EDA token + future state).
3. Back up `inventory/local/vault.yml` separately from the git repo (currently bundled with git, but vault password is not).
4. Restore any of the above onto a fresh hub.
5. Test the restore. (No "verify your backup actually works" recipe.)

**Recommended new section**: `03 §5 Backup & Restore` (English, in `03-configuration.md` since the topic is "managing your config + state"). Outline:

- §5.1 What to back up (3 categories: code = git; secrets = vault + .vault_pass + .env; state = `/var/lib/ansispire/state/` + audit DB volume)
- §5.2 How to back up (each category — concrete command lines)
- §5.3 How to restore (per category, with the order-of-operations dependencies)
- §5.4 How to verify a restore (a smoke recipe similar to `01 §5.4`)
- §5.5 Backup hazards (encrypted-vs-not, what NOT to put into the same backup blob, retention)

Size estimate: ~80–120 lines. Sized to fit within `03`; no new file needed.
