**Verification methodology applied** (recorded so future rounds can re-use):

1. Grep every `make <target>` reference in edited docs against `^<target>:` in Makefile — all 18 targets verified.
2. Read each compose file referenced; cross-check container names + volume names + project name (via `docker compose config`).
3. Read `reactor.py` for actual log strings before citing them in verification recipes.
4. Verify test counts (`grep -c '^def test_'`) against `02 §9` claim — 14 + 9 + 5 = 28 ✓.
5. Sweep for stale section refs (`02 §10`, `02 §12`, `02 §13`) anywhere in docs — none remain outside `reviews/`.
6. Sweep for the just-fixed wrong-prefix pattern (`controller_semaphore-data`, `controller_audit-data`) — zero remaining occurrences.

**Lesson** (added to feedback memory): for any new content describing **executable** commands (docker, make, log greps), read the source-of-truth file before drafting, not after. Structural self-checks (line counts, section TOC) do not catch executable-string-content bugs.
