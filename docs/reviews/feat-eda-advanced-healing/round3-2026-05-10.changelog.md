## 3. 每项变更的意图

### P3.1 — events.schema.json + reactor schema_banner
- **为什么**：上一轮 Gemini 重构 rules.json 引用 `description_contains` 没问题（运气），但 rules ↔ event 字段是隐式契约；任何一方漂移都没有第三方定义可指。给 schema 一个 `$id` + `version`，把"事件长什么样"显式化。
- **为什么打 banner**：reactor 启动时打印 `event schema: <$id>@<version>`，运维一眼能看到当前栈用的是哪个版本——契约升级后能从日志做证据。

### P3.2 — rule.enabled 软关
- **为什么**：DB Failover rule 引用 `playbooks/remediation/db_failover.yml`，但目前是 placeholder（debug-only）。删掉 rule 会让 contract test 失败；保留并开启会在 e2e 触发时跑一个不打算落地的 playbook。引入 `enabled: false` 提供"先在 rules.json 占位，等 playbook 真做完再打开"的中间态。
- **`_disabled_reason` 字段**：未来打开时第一件事是回头读这条注释，避免"为什么 disabled 没人记得"。
- **`match_rule` 早返回**：单元测 `test_disabled_rule_never_matches` 把行为锁死。

### P3.3 — L2 C9 schema-vs-rules 对齐
- **为什么**：上一轮 contract test 只挡住了 rules ↔ bootstrap template-name 漂移（C7）。如果未来谁在 rules.json 加 `severity_contains: "high"` 但 schema 里没声明 `severity`，reactor 会静默永不 MATCH——无错误、无日志、纯坑。C9 在 PR 阶段直接 fail。
- **`_contains` 剥离**：先按 rules.json 当前唯一支持的 operator 写；演化预案在 spec §8 记好了，将来引入新 operator 这里要扩展。

### P3.4 — L4 TSVS spec 先于代码（spec-first）
- **为什么**：上轮反馈"是否应该先做一个测试文档，可以清晰的做出可以使用的命令操作，以及预测结果。然后用实际结果对比来确定代码是否有问题"。Phase 3 全程按 spec → 代码 → 跑 → 回填执行。L4 spec 在 P3.4 写完后才动 P3.5。

### P3.5 — L4 e2e harness
- **隔离决策**：
    - 项目名 `ansispire-e2e`、网络 `controller-net-e2e`、容器 `*-e2e` 三层隔离；
    - 服务名复用 dev (`semaphore` / `audit-sink` / ...) 配合 network alias，让 reactor.py / relay.py 默认 `http://semaphore:3000` 等无需改 env；
    - 卷无 top-level name，`down -v` 真清理。
- **bootstrap secrets_path 参数化**：让 e2e 把 token 写到自己的 `.secrets`，不污染 dev 的 `controller/semaphore/.secrets`。
- **端口选择**：原 spec 设计 3310/3320 (+10 偏移)，但 dev audit-sink 占 3310 → 修订为 3320/3330 (+20)。spec §2 已注明历史。
- **run.sh 7 步 + EXIT trap**：success / failure / Ctrl-C 都跑 `down -v`；`E2E_KEEP=1` 留现场。

### P3.6 — 跑 e2e + 回填
- **首跑结果**：`make test-eda-e2e` 总耗时 57 s（< 90 s 预算），task 1 status=success in 23 s。
- **隔离验证**：teardown 后 e2e 容器/卷/网络全无残留；dev 4 容器 (`ansispire-semaphore` / `ansispire-audit-{sink,relay,reactor}`) 全程不受影响；dev `.secrets` 未被改写。
- **TSVS §6/§7 已用实测命令输出回填**。
