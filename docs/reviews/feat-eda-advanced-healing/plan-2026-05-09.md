## 3. Architectural Decisions

### A. Port Convention (project-wide)

**Decision**: every Ansispire-owned service that binds an HTTP listener uses host port = `containerPort + 300` for the default, encoded in `.env.example` and role defaults.

| Service | Container port | Host default | Source of truth |
|---|---|---|---|
| Semaphore | 3000 | **3300** | `controller/semaphore/.env.example:SEMAPHORE_PORT` ; mirrored in `roles/ansispire_hub/defaults/main.yml` |
| Audit Sink | 3010 | **3310** | `controller/audit/docker-compose.yml:AUDIT_PORT` (already `${AUDIT_PORT:-3010}`, default flips to 3310; loopback bind unchanged) |
| Reactor | none | none | (no listener, network egress only) |
| Relay | none | none | (no listener, network egress only) |
| (future) Prometheus | 9090 | **9390** | reserved by this convention |
| MySQL | 3306 | 3306 | **excluded** — IANA-standard protocol port, not project-owned |

Container-side ports stay original (so the Semaphore image's internal `3000` is unchanged); only the host-exposed mapping is offset. This way upstream image upgrades don't require port edits.

A short table goes into `SUMMARY.md` under "Network Map".

### B. Authentication Standard

**Decision**: Bearer Token only for M2M; password is bootstrap-time only and never persisted by reactor/relay.

Bootstrap flow (re-codified in bootstrap.yml):

```
[1] Cookie login (admin/password, one-time)
[2] GET /api/user                              → discover self user_id
[3] POST /api/user/<id>/tokens                 → mint long-lived M2M token
[4] Persist token to controller/semaphore/.secrets (gitignored)
[5] All subsequent provisioning (project, repo, inventory, env, templates)
    uses Authorization: Bearer <token>          ← no more cookies
[6] audit compose loads --env-file ...secrets   ← reactor & relay get token
```

`controller/semaphore/.secrets` keeps the token isolated from `.env` (which is the user-edited config file). `.gitignore` already excludes `*.secrets` patterns — verify and extend if needed.

### C. EDA Test Stratification (the "decoupled testing" you asked for)

The point is: **rules.json edits should be testable without bringing up Semaphore**. Today there's no such layer. We introduce four:

| Layer | Name | Inputs | External deps | Validates | Make target | Expected wall time |
|---|---|---|---|---|---|---|
| **L1** | Unit | mocked event dicts | none | `match_rule` boolean logic, `_contains` semantics, cooldown timing | `make test-eda-unit` | <2s |
| **L2** | Contract | real `rules.json` + fixture events | none | (a) JSON schema valid, (b) every `template_name` referenced exists in `bootstrap.yml`, (c) every event fixture parses through `process_event` to the expected rule, (d) reactor's view of `payload.event.*` matches the schema relay claims to emit | `make test-eda-contract` | <3s |
| **L3** | Component | reactor container + mock Semaphore HTTP stub | python `http.server` only | reactor's outbound HTTP request is exactly the contract: method, path, headers (Bearer), JSON body shape — verifies `trigger_semaphore_task` end-to-end without Ansible | `make test-eda-component` | <15s |
| **L4** | E2E | disposable compose project | docker | full chain: synthetic jsonl line → reactor → real Semaphore (test instance) → template → no-op playbook (`connection: local`, `debug` only) | `make test-eda-e2e` | <90s |

**Why each layer is necessary**:
- **L1** catches reactor logic regressions (Gemini-style refactors).
- **L2** catches the failure mode that's biting us today: rules.json drifts away from bootstrap's templates and the only feedback is a runtime "could not resolve template" log entry. Should fail at PR time.
- **L3** catches outbound API contract drift (e.g., Semaphore upgrade changes `/api/project/<id>/tasks` payload). Runs in CI without docker.
- **L4** is the truth oracle. Lives in project smoke; not gated in CI, but `make verify-full` adds it.

L1 already partially exists (`controller/audit/test_reactor.py` after Gemini's edit); we extend it. L2/L3/L4 are new.

`events.schema.json` (new file under `extensions/eda/`) is the contract artifact; reactor.py and relay.py both load and validate against it on startup (warn-only at first), and L2 enforces strictly.

### D. bootstrap.yml: revert + extend, not refactor

Per W-R13 minimum-modification principle for audits: the destructive rewrite from 486 → 158 lines is reversed. We `git checkout HEAD -- controller/semaphore/bootstrap.yml` and **append** to it:

- the token mint flow (B above)
- the two remediation template registrations (currently in the broken rewrite)

This keeps the password assertion, no_log, idempotency guards, and clear error reporting that the original had. Net change should be **+~80 lines, not -328**.

### E. Event Schema Alignment

After reading `controller/audit/sink.py` directly, the picture is:

- **sink** wraps every POST as `{"ts","remote","path","ua","payload"}`, with the raw HTTP body in `payload`
- **relay** POSTs `{"source": "semaphore", "event": <ev>}` → sink writes `{"ts":...,"path":"/event",...,"payload":{"source":"semaphore","event":<ev>}}` to jsonl
- **reactor** reads `payload["payload"]["event"]` from each jsonl line
- **manual injection** (`printf '{"payload":{"event":...}}' >> events.jsonl`) bypasses sink, writes the inner shape directly — and reactor still sees `payload.payload.event`

So today **the chain works by accidental double-wrap**, not by contract. Two writers (relay-via-sink vs. direct injection) reach the same key path through different mechanisms. Any well-meaning change to sink, relay, or "what shape do we write directly" silently breaks one branch.

We freeze it: author `extensions/eda/events.schema.json` defining the *inner* event shape (the `payload.event.*` envelope reactor reads); make sink/relay/injection-fixtures all conform; reactor logs schema version on startup and warns on mismatch. L2 contract test fails on schema drift.

### G. Port Single-Source-of-Truth

**User requirement at approval (Q7)**: ports must live in one variable file, not be sprinkled across `.env`, role defaults, inventory vars, and compose files.

Inventory in this repo does **not** have a top-level `group_vars/`; each environment (`dev` / `staging` / `production`) has its own under `inventory/<env>/group_vars/`. So Ansible's auto-load mechanism is per-environment. The clean SSOT design:

```
config/ports.yml                       ← SSOT (the only place a port number lives as a literal)
├── consumed by Ansible:    via vars_files: "{{ playbook_dir }}/../config/ports.yml"
└── consumed by Compose:    via Make target `ports-sync` which renders config/ports.yml → controller/semaphore/.env (port lines only)
```

`config/ports.yml` shape (initial):
```yaml
---
# Single source of truth for all Ansispire-owned host ports.
# +300 offset from container port to avoid common collisions (3000 web,
# 5432 pg, 6379 redis, 9090 prom, etc.).
# Edit this file; run `make ports-sync` to propagate to .env.
ports:
  semaphore_host: 3300        # → container 3000
  semaphore_container: 3000
  audit_sink_host: 3310       # → container 3010 (loopback bind only)
  audit_sink_container: 3010
  # reserved for future use:
  prometheus_host: 9390
  prometheus_container: 9090
```

Make targets:
- `make ports-sync` — read `config/ports.yml`, regenerate the port-related lines in `controller/semaphore/.env` (preserves user-edited admin password, etc.); idempotent
- `controller-up` depends on `ports-sync` so the .env is always in lockstep
- `controller-bootstrap` reads `config/ports.yml` directly via `vars_files`, so there's no chance of cross-file drift

Role defaults (`roles/ansispire_hub/defaults/main.yml:4` etc.) are kept, but become **derived** from `config/ports.yml`: hub role's `ansispire_hub_port` defaults to the same value, and inventory `vars.yml` no longer hardcodes 3300 — it inherits.

Acceptance: `grep -rn "3300\|3000\|3310" --include='*.yml' --include='*.j2' --include='Makefile'` after Phase 1 returns only references inside `config/ports.yml`, schema files, and Make-rendered output (`.env`); no other source carries a port literal.
