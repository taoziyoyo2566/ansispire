## 3. 每项变更的意图

### S1 — Manifest SSOT (config/manifest.yml)
**问题**：版本号四处写死（dev compose `:v2.18.2`，hub role template `:latest`，e2e compose `:v2.18.2`），ports SSOT 只覆盖 Path B。
**做法**：单文件管所有；版本采用 `default_tag: latest` + `*_pinned` 显式字段 + `semaphore: pinned ?? default` 的解析键，让"用户可以选择跟 latest，也可以钉版"两种模式都成立。
**两条路怎么读**：Ansible 走 `vars_files: config/manifest.yml`；Compose 走 `manifest-sync` 渲染 .env 后用 `${VAR}` 插值。

### S2 — Inventory taxonomy
**问题**：`[hub]` 单组写死了远程节点；用户的实际生命周期是「先本地起 → 后迁远程 → 可能并存」；下一阶段还要加 4 台不同 OS 的 target VPS。
**做法**：拆 `hub_local` / `hub_remote`，再用 `[hub:children]` 合并；切换全靠 `--limit`（封装成 `make hub-deploy NODE=local|remote|all`）。同时把未来 `[targets_debian/rhel/alpine]` 的占位也写好——空组无害但锁定了命名约定。

### S3 — Hub role：去模板化、迁状态、夯 rsync
**Path A 真要能用**，要 5 件事一起：
1. **去 compose 模板**：`semaphore_docker_compose.yml.j2` 是 Path B 已修过的 compose 的一个不同步副本（用 `:latest`、写死 `extra_hosts: ans-hk01:host-gateway`、缺 healthcheck）。删掉，让 rsync 直接把 Path B 的 SSOT compose 落到部署目标。
2. **状态文件迁出代码目录**：`.eda_token` 原来在 `/opt/ansispire/.eda_token`（rsync target dir），每次 `delete: true` 都把它擦掉、下次再 mint 一个新的 → 旧 token 失效，整个 audit 链路在生产里实际上是 broken-by-design。迁到 `/var/lib/ansispire/state/.eda_token`，rsync 看不见就不会动。
3. **rsync 强 exclude**：原来 `--exclude=.git --exclude=.venv` 留了大量漏洞——`.claude/settings.local.json`、`controller/rbac/.demo_*.pw`、`controller/rbac/users.yml`、`controller/semaphore/.env`、`.secrets`、`controller/audit/e2e/.env`、`__pycache__` 全在飞向远程主机。这是安全问题，不是优化问题。
4. **--check mode 守护**：Provision Admin/Token 块内的 `cookies_string` 在 --check 模式下因 Login 任务 noop 而不存在 → 整块加 `when: not ansible_check_mode`；Load/Set token 加 token 存在性 gate；audit role 的 docker_compose_v2 加 `ansispire_hub_eda_token is defined or not ansible_check_mode`。
5. **rsync 输出对比验证**：T5 跑出来的 diff，前面那些泄密 / 删 token 的行**全部消失**——这条改动有据可查。

### S4 — infra_baseline OS 守门
**为什么现在做**：用户已宣布下一阶段 4 台 VPS 包括 Alpine 和 Rocky。如果不守门，role 在 Alpine 上跑到 `apt:` 直接 fail，错误信息还不直观。
**做法**：apt 段套 `when: os_family == "Debian"`；RHEL/Alpine 段写显式 `fail` 占位（带具体怎么实现的提示）——比静默 skip 好（静默 skip 会让人以为部署成功了但实际啥都没做）。

### S5 — Makefile hub-deploy 包装
让 `--limit` 不再是用户要记的语法。`NODE=local|remote|all` 三选一，外加 `VAULT_PASSWORD_FILE` 默认 `.vault_pass` 但可覆盖。`hub-deploy-check` 是 `--check --diff` 版本。
